# 6. نظام المشتريات (Purchase System)

## 1. العمليات المدعومة
- طلب شراء داخلي (طلب جرد، طلب صلاحيات)
- فاتورة شراء (استلام أصناف، تحديث مخزون)
- مرتجعات شراء حالة وجود بضاعة معيبة أو زائدة
- تتبع دفعات الموردين والأرصدة

## 2. دورة الشراء (شراء نموذجية)
```
[طلب شراء] -> [موافقة مدير] -> [إصدار أمر شراء] -> [استلام بضاعة] -> [تسجيل فاتورة] -> [تحديث مخزون ومحاسبة ومورد]
```

## 3. الربط مع المحاسبة والمخزون
- كل عملية شراء تُولد قيد محاسبي وتُحدث المخزون وتعدل حساب المورد

# 7. نظام المخزون (Inventory System)

## ميزات رئيسية
- إدارة المنتجات عبر وحدات وباركود متعدد
- تنظيم الأصناف والفئات والوحدات (علبة/كرتون/حبة ...)
- متابعة الجرد آليًا وحد إعادة الطلب وتنبيهات نفاد الأصناف
- تتبع حركة مخزون دقيقة (دخول/خروج/تحويل)
- إدارة مخازن وفروع بلا حدود

## كيان المنتج
| الحقل             | الوصف                                |
|-------------------|--------------------------------------|
| اسم المنتج        | مثال: طحين فاخر                      |
| الرمز & الباركود  | يدعم باركودات متعددة                 |
| الوحدة الافتراضية | كرتون / حبة / كجم                    |
| التصنيف           | بقالة / عصائر / أجبان                |
| الحد الأدنى       | ينبه عند الوصول إليه                  |

## دورة الجرد وحركة المخزون نصيًا
```
[إضافة أصناف] -> [بيع/شراء] -> [تحديث رصيد مخزون] -> [تنبيه جرد/نفاد] -> [جرد دوري/تصحيح]
```

# 8. نظام العملاء (Customer System)

## ميزات رئيسية
- قاعدة بيانات شاملة للعملاء (بيانات اتصال – تصنيف – فروع مرتبطة)
- كشف حساب آني للعميل
- متابعة الديون والتنبيهات الذكية عبر SMS
- تقارير أداء العملاء، والمنتجات الأكثر مبيعاً لكل عميل
- تصنيف للعملاء (حسب النشاط/المنطقة/الأرصدة)

## العمليات الأساسية
| العملية          | الهدف                        | نتيجة التكامل             |
|------------------|------------------------------|---------------------------|
| إضافة عميل       | فتح جديد مع بيانات كاملة     | تحديث الحسابات والمبيعات  |
| تسديد دين        | ربط بحركة محاسبية وفاتورة    | تقليل رصيد، SMS للعميل    |
| كشف حساب         | عرض كل العمليات             | تقرير فوري وتاريخي        |
| إرسال SMS        | تنبيه بالدين/تذكير/عروض      | تقليل المديونية           |

---

# 9. نظام الموردين (Supplier System)

## ميزات رئيسية
- إدارة قاعدة بيانات الموردين (اسم/هاتف/فئة/عناوين)
- متابعة كشف حساب المورد بكل تفاصيل الفواتير والمدفوعات
- تقارير متقدمة بأرصدة الموردين والمديونيات
- دعم المرتجعات والتحصيل الجزئي/الكامل
- ربط كل فاتورة شراء بحساب المورد تلقائياً

## العمليات الأساسية
| العملية        | الشرح                             | تكاملها مع النظام |
|----------------|-----------------------------------|-------------------|
| إضافة مورد     | بيانات تفصيلية وربط بالمنتجات      | تحديث محاسبة      |
| دفع لمورد      | ربط القيد والحساب والفاتورة        | تحديث رصيد        |
| مطالبة مورد    | إرسال إشعارات للموردين            | تسوية أوتوماتيكية |

# 10. نظام التقارير (Reporting System)

## أنواع التقارير
| التقرير            | محتواه                          |
|--------------------|----------------------------------|
| تقارير المبيعات     | حسب الفترة/الفروع/الأصناف       |
| تقارير الأرباح      | مجمل وهامشي وتحليل أسباب الأرباح |
| تقارير المخزون      | جرد – دوران – أصناف راكدة       |
| تقارير العملاء      | كشف حساب – الديون – الأداء       |
| تقارير الموردين     | كشوفات – أرصدة                  |
| تقارير الأداء Dashboard| رسوم بيانية، مؤشرات حية        |

## ميزة Dashboard حديثة
- رسوم بيانية تفاعلية (Bar, Pie, Line)
- Top Products/Fastest Moving Items
- مؤشر مبيعات يومي/شهري فوري
- مؤشرات تسهيلات وديون متأخرة
- دعم تصدير التقارير PDF/Excel
